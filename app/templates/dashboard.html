Railway needs to know its own public URL to build webhook callbacks correctly.

  1. Open Railway → your service → Variables
  2. Add a new variable: BASE_URL
  3. Set its value to your Railway public URL (shown below)
  4. Railway will automatically redeploy

Step 2 — Configure Twilio Phone Number

Paste these URLs into your Twilio Console so calls are routed to this app.

  1. Go to console.twilio.com → Phone Numbers → Manage → Active numbers
  2. Click your Twilio number
  3. Scroll to "Voice & Fax"
  4. Set A call comes in to Webhook and paste the Inbound URL below
  5. Set method to HTTP POST
  6. Optionally set Call Status Changes to the Status URL
  7. Click Save configuration

Step 3 — Twilio Trial Account Note

On a Twilio trial account, you can only call verified phone numbers (numbers you've verified in the Twilio console). Calls to unverified numbers will fail.

To verify a number: Twilio Console → Phone Numbers → Verified Caller IDs → Add a new Caller ID.

Trial calls also begin with a Twilio announcement — this is removed on paid accounts.
